# Case Study: Billi

Billi is a personal finance app designed for the Mexican market — specifically for university students aged 18-25 who want to track daily expenses without the complexity of traditional budgeting tools.

Billi was the first project we scaffolded end-to-end using the prototype-kit workflow described in this Playbook. A non-developer student wrote the three required documents (`product-brief.md`, `ux-vibes.md`, and `screens.md`) in one afternoon, ran `/prototype-from-docs`, and had a working React prototype with five screens by the end of the same day.

## What we learned

**Specificity in `product-brief.md` pays off.** The first iteration used vague language like "an easy app for budgets." The second iteration named the target user explicitly — "Mexican university students with irregular income" — and the generated UI immediately felt more appropriate: Spanish labels, peso sign, weekly cadence instead of monthly.

**`ux-vibes.md` is the highest-leverage document.** Three reference sites and a two-sentence tone description produced dramatically better visual output than two pages of color hex codes. The agent interprets mood better than specifications.

**The fallback path matters.** When the agent hit an ambiguous screen description, it asked a clarifying question rather than guessing. Teaching students to expect and welcome those questions reduced frustration and improved output quality.
